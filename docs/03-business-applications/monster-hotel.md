# Monster Hotel

**Location:** `03_Business_Applications/MonsterHotel`

## Overview

Monster Hotel is an advanced hotel management system with a complete MVC architecture for managing monster guest bookings. The application features a sophisticated booking system, persistent data storage, statistical analysis, and HTML report generation. It demonstrates enterprise-level design patterns for hospitality management systems.

## Key Features

- Create and manage bookings with guest species classification
- Support multiple room types for different guest sizes
- Implement booking repository for data persistence
- Service layer for business logic separation
- Generate statistical reports on bookings
- Export bookings to HTML format
- Load and save booking data from/to files
- Complete menu-driven wizard interface
- Guest search functionality
- Financial and occupancy statistics

## Technologies Used

- MVC architectural pattern (Model-View-Controller)
- Repository pattern for data access
- Service layer for business logic
- Enumeration types (Species, RoomType)
- File I/O with data persistence
- Template-based HTML generation
- Statistical data aggregation
- DateTime handling with LocalDate
- Console-based wizard interface

## Learning Objectives

This project helps to learn to build enterprise-level hotel management systems using MVC architecture, implement repository patterns for data persistence, separate business logic into service layers, generate statistical reports, and design sophisticated multi-feature hospitality applications.
